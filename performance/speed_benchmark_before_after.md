## 5. 哪些优化最有效

### 5.1 最有效：文本生成链路
- 原来最大的慢点不是文本逻辑本身，而是缩进 JSON 输出。
- 改成紧凑流式写出后，JSON 编码/写盘成本显著下降。
- 再叠加 label 解析并发，整体从 `72s+` 下降到 `9s+`。

### 5.2 第二有效：模型文本快路径
- 文本哈希从“逐条创建小 tensor + 逐样本补缺”改成“整句缓存 + 批量去重 + 统一特征构建”后，模型侧文本准备热点直接下降一个数量级。
- `TextVoxelNet`、`TripleModalBEVFusion`、`TripleModalRLBEVFusion` 共享同一条快路径，避免了三处重复维护慢实现。

### 5.3 第三有效：infos 构建并发化
- 预扫描 sample-folder 索引后，去掉了每样本重复 folder 探测。
- 图像尺寸、标定、标签解析改成线程并发后，小文件 I/O 等待时间被明显摊薄。

### 5.4 训练侧最关键：单 token 注意力和投影快路径
- RL 融合里的单 token 文本注意力原本走完整 `MultiheadAttention`，现在改成等价快速路径后，单热点提升约 `42x`。
- 图像投影到 BEV 通过投影模板缓存、齐次坐标缓存和更合理的采样路径，当前微基准提升约 `1.24x`。

## 6. 正确性检查
- `radar_converter` 前后样本计数一致：`train=5139, val=1296, test=2247`
- `radar_text_converter` 前后记录数一致：`train=5139, val=1296, train_val=6435, full=8682`
- 单 token 文本注意力新旧路径最大绝对误差：`1.1920928955078125e-06`
- 图像投影到 BEV 新旧路径最大绝对误差：`0.0`
- 优化未删除字段、未修改类别语义，只改变了执行路径、缓存方式与输出格式。

## 7. 结果解释
- prompt 生成提速不如前两项明显，是因为它本来就不是主要瓶颈，且当前耗时主要由 JSON 读取主导。
- 模型侧热点提速没有直接等价为完整训练 step 的端到端倍数，因为当前环境仍缺少完整训练回归；但从前向热点位置看，这些优化会真实减少每 step 的 Python 开销和辅助融合开销。
- 图像投影到 BEV 的收益没有文本快路径那么夸张，是因为这部分本身就更接近算子密集路径，能优化的主要是“准备成本”和“多余张量构造”。
